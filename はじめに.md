# おうちの買い物 ・ セットアップ手順

家族でリアルタイム共有する買い物・在庫リスト（単一HTML＋Firestore＋PWA）。
コードの編集は不要です。以下の準備をして、フォルダごとNetlifyに公開するだけです。

---

## 1. Firebase プロジェクトを作る（無料）

1. https://console.firebase.google.com/ で「プロジェクトを追加」
2. **Firestore Database** を作成（本番モードでOK。リージョンは asia-northeast1 など近い場所）
3. **Authentication** を開き「ログイン方法」で **匿名** を有効化
4. Firestore の「ルール」タブに、同梱の `firestore.rules` の中身を貼り付けて「公開」
5. プロジェクト設定（歯車）＞「アプリ」で Web アプリ（`</>`）を追加
   → 表示される `firebaseConfig = { ... }` を**まるごとコピー**（後でアプリに貼ります）

## 2. Netlify に公開する

1. https://app.netlify.com/drop を開く
2. この **`kaimono-app` フォルダごと**（または `kaimono-app.zip`）をドラッグ＆ドロップ
3. 発行された URL（例：`https://xxxx.netlify.app`）が公開アドレスです

## 3. 承認済みドメインを追加（重要）

Firebase の **Authentication ＞ Settings ＞ 承認済みドメイン** に、
Netlify のドメイン（`xxxx.netlify.app`）を追加します。
※ これを忘れると「未接続」のままになります。

## 4. アプリを開いて初期設定

公開URLを開くと初回設定画面が出ます。
- STEP1：コピーした `firebaseConfig` を貼り付け
- STEP2：あなたの名前・色・**家族コード**（例：`yamada-home`）を入力

「はじめる」で完了。家族には**同じ公開URLと同じ家族コード**を伝えれば、同じリストに繋がります。
各自のスマホで「ホーム画面に追加」すると、アプリのように使えます。

---

## できること
- 買い物リスト（売り場順カテゴリ・履歴/定番からワンタップ追加・誰がいつ）
- 在庫の3状態（ある／少ない／切れた）。「切れた」で買い物リストへ自動追加
- 買ったらチェック → 同名の常備品は自動で「ある」に戻る
- 更新の履歴 ／ JSONバックアップ ／ お使い依頼

## まだ有効化していないもの（後で追加）
- **プッシュ通知の配信**：設定の「通知の鳴らし方」は保存されますが、実際にスマホへ
  通知を飛ばすには OneSignal などの登録が必要です。準備できたら組み込みます
  （`index.html` 内の `pushStub()` が差し込み口）。それまでも、リストはリアルタイムに同期します。

## 家族コードについて（安全面）
このアプリは「コードを知っていれば見られる」方式です。買い物リストなので機密性は
低いですが、コードは推測されにくい言葉にし、家族以外には共有しないでください。
